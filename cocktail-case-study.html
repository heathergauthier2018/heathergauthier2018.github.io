Cocktail Finder — Random Generator + Search Explorer

Cocktail Finder is a single-page web application that helps users explore cocktails in two ways: generating a random drink or searching for something specific. Favorites are saved locally, allowing users to build a personal list without creating an account.

The project is an active work in progress; the live demo reflects the current iteration while I continue refining the interface, improving the mobile experience, enhancing accessibility, and tightening the code structure.

Why I Built This

I created Cocktail Finder to strengthen my JavaScript fundamentals while working with a real public API. I wanted a project that would let me practice client-side state management, DOM updates, and building a small single-page experience without relying on a framework. It also gave me a chance to make concrete UI and accessibility decisions on a project I fully own from idea to deployment.

Features

Cocktail Finder is designed to feel like a small, focused tool instead of a sprawling site. Users can:

  • Generate a fully detailed random cocktail, including image, ingredients, glass type, and instructions.
  • Search for cocktails by full or partial name using TheCocktailDB API.
  • Save cocktails to a personal favorites list powered by localStorage.
  • Use a clean, desktop-first layout, with mobile refinements planned as a future enhancement.

Random Cocktail Mode

One click triggers an API call to TheCocktailDB and displays a new drink with its name, image, glass type, ingredients, and step-by-step instructions. This mode is meant to encourage discovery when the user doesn’t know what they’re in the mood for.

Search Mode

Users can search by cocktail name, even with partial matches. Results are shown as a list of cards that highlight key details and can be favorited for later. This supports both revisiting known drinks and exploring new ones by name.

Favorites System

Favorited cocktails are stored in localStorage, so they persist across sessions on the same device. Users can build a simple “go-to” list without needing a back-end, account system, or sign-in flow.

Layout & UI

The current layout is optimized primarily for desktop browsers with clear hierarchy and spacing. Improving the mobile layout and adding more refined responsive behavior is part of the planned next round of enhancements.

Technical Decisions & My Role

This project started as a way to practice working with a public API and managing state in a small vanilla JavaScript codebase, and it grew into a focused single-page app where I owned both the technical and UX decisions.

Technical Decisions

  • Structured the app as a single-page experience, toggling Random and Search views on one HTML page instead of loading separate routes.
  • Used fetch() to integrate TheCocktailDB API and added basic error handling and empty-state messaging for missing or invalid results.
  • Implemented favorites via localStorage so users can revisit saved cocktails without an account or back-end.
  • Separated DOM update logic from data formatting to keep the JavaScript easier to follow and maintain as the project evolves.
  • Focused on clear typography, contrast, and keyboard-focus states to improve accessibility for keyboard and screen-reader users.

My Role & Stack

  • Designed the overall user flow for Random, Search, and Favorites.
  • Built the UI with semantic HTML and custom CSS.
  • Wrote all JavaScript for view switching, API requests, and favorites management.
  • Deployed the project using GitHub Pages and iterated based on manual testing.

Tech Stack

  • HTML, CSS, Vanilla JavaScript (ES6+)
  • TheCocktailDB API

Future Enhancements

Cocktail Finder is intentionally small, but there’s plenty of room to grow it into a more powerful, still-approachable cocktail tool. These ideas focus on better discovery, organization, and usability, especially on smaller screens.

Advanced Filtering

Add filtering by ingredient, category, or glass type so users can narrow down results more efficiently. For example, they could view only “whiskey” drinks in a rocks glass or browse light, fruity options.

Ingredient-Based Navigation

Enable ingredient-based navigation (e.g., clicking “lime” or “rum” on a cocktail card shows other cocktails that include that ingredient), turning each drink into a jumping-off point for exploring the rest of the API.

Tags & Favorites Organization

Introduce tags or labels (like “party”, “summer”, or “cozy”) for organizing favorites or grouping cocktails by theme, making it easier to build go-to lists for different moods or occasions.

Surprise Me (Within a Category)

Add a “surprise me within a category” option so users can ask for a random mocktail, a summer drink, or a tequila-based cocktail and get more targeted recommendations than the global random mode.

Mobile Polish & Micro-Animations

Improve the mobile layout and spacing, and add subtle animations or transitions for card loading, favorites, and view switching to create a more polished feel and better usability on smaller screens.

Non-Alcoholic Mode

Add a non-alcoholic mode toggle that filters results down to mocktails and low- or no-alcohol drinks, making Cocktail Finder more inclusive for users who don’t drink alcohol or want better alcohol-free options.
