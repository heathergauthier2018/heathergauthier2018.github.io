Why I Built This Site

This project was completed as part of WGU’s D277 Front-End Web Development course. The assignment asked students to create a multi-page informational site that demonstrates core front-end development skills without using frameworks.

I chose Washington State as the subject and built the entire project from scratch, focusing on semantic structure, responsive layout, and accessibility. Once the assignment was complete, I felt the final product clearly represented my strengths in foundational front-end skills, so I deployed it to GitHub to showcase it in my portfolio as-is.

Project Overview & Experience

The site is organized across multiple pages, each focusing on a different aspect of Washington State (overview, regions, attractions). It is intentionally simple, clear, and informational—mirroring the structure of real-world tourism or educational sites.

  • Multi-page architecture. Each page has a clear purpose and uses consistent navigation so users always know where they are.
  • Fully semantic HTML. Elements like <header>, <nav>, <main>, and <footer> create an accessible and meaningful structure.
  • Readable content layout. Text is organized with hierarchy, spacing, and imagery to support scanning and comprehension.
  • Alt-tagged imagery. Every image includes thoughtful alt text for screen readers.

Layout & Visual Design

The site uses CSS Flexbox and Grid to create a flexible, responsive layout that adapts to mobile, tablet, and desktop screens. Visual decisions emphasize clarity and stability.

Responsive Grid

Cards and page sections rearrange fluidly to remain readable on all screen sizes.

Consistent Visual Rhythm

Typography, spacing, and layout styles are applied consistently to create a unified feel.

Accessible Color Choices

Colors were chosen with contrast and readability in mind, following accessibility best practices.

Navigation & Feedback

Simple navigation patterns and hover cues help guide the user through the site.

Interactivity & Accessibility

JavaScript was used only where necessary to add small enhancements. Because the project emphasizes fundamentals, most of the site is powered by clean HTML and CSS.

  • Small interactive elements improve responsiveness and feedback.
  • Keyboard-friendly navigation and focus states support accessibility.
  • Meaningful alt text and ARIA attributes help screen reader users.
